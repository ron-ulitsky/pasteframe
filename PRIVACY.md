# PasteFrame Privacy Policy

Last updated: May 19, 2026

PasteFrame is a Chrome extension that helps users paste images into Google Docs
comments by uploading those images to the user's Google Drive and inserting a
Drive link into the comment.

## Data PasteFrame Handles

PasteFrame handles only the data needed to provide this feature:

- Images the user explicitly pastes into a Google Docs comment or reply field.
- The generated Google Drive file metadata needed to insert a link, such as file
  ID, file name, and Drive URL.
- Extension settings, such as the Drive folder name and link sharing preference.

PasteFrame does not collect browsing history, document text, analytics events,
or advertising identifiers.

## How Data Is Used

When a user pastes an image into a Google Docs comment, PasteFrame:

1. Reads the pasted image from the clipboard.
2. Uploads the image to the user's Google Drive.
3. Stores uploaded images in a `PasteFrame` folder by default, unless the user
   changes the folder name in extension options.
4. Inserts the resulting Google Drive link into the comment field.

PasteFrame uses the Google Drive API only to create and manage files and folders
needed for this feature.

## Data Sharing

PasteFrame does not sell, rent, or share user data with third parties. Uploaded
images are stored in the user's own Google Drive account.

If the user enables link sharing, PasteFrame changes the uploaded file's Drive
permissions so anyone with the link can view that image. Users can disable this
setting from the extension options page.

## Data Storage

PasteFrame stores extension settings using Chrome extension storage. Uploaded
images are stored in the user's Google Drive. PasteFrame does not operate a
separate backend server.

## Data Deletion

Users can delete uploaded images directly from Google Drive. Users can also
remove PasteFrame's local settings by uninstalling the extension or clearing
extension data in Chrome.
